# References ‚Äî paper memory

Curated summaries of papers relevant to this repo, for humans and agents
iterating on `eb_jepa`. One folder per paper: `paper/<slug>/<slug>.pdf` +
`paper/<slug>/SUMMARY.md`. PDFs are git-LFS-tracked automatically
(`paper/.gitattributes`); extracted `*.txt` sidecars are git-ignored.

## Add a paper

1. Fresh clone? Run `git lfs install` once.
2. `mkdir references/paper/<slug>` ‚Äî short, lowercase, hyphenated.
3. Download the **full latest** arXiv version and verify it is not truncated:
   ```bash
   curl -sL -o references/paper/<slug>/<slug>.pdf https://arxiv.org/pdf/<arxiv-id>
   python3 -c "import fitz; d=fitz.open('references/paper/<slug>/<slug>.pdf'); print('pages:', d.page_count); print(repr(d[d.page_count-1].get_text()[-200:]))"
   ```
   The last page should end on the references/conclusion. **Do not trust
   `file`'s page count** (it undercounts nested page trees); the bare
   `/pdf/<id>` already serves the latest version. If the paper is short with no
   appendix that may be correct (ICME, Graz BCI, etc. are ~6 pages); if it looks
   cut off, the full version may be OpenReview-only.
4. Write `references/paper/<slug>/SUMMARY.md` using the skeleton below (no need
   to commit a `.txt`; extract on demand with `python3 -c "import fitz; ..."`).
5. Add a row to the appropriate tier table below, then
   `git add references/paper/<slug>` and commit. The PDF is stored in LFS
   automatically.

## Papers

Grouped into a **Core** set (the `eb_jepa` codebase plus the vision/robotics
world-model line it builds on) and three maturity tiers for the
JEPA-to-new-modality literature. Each row carries a modality/track tag.

### Related world models (non-JEPA)

Pixel-space / diffusion world models included for comparison: they share the
action-conditioned planning setup (CEM/MPC over imagined rollouts) but predict
in pixel space rather than in a learned JEPA latent, so they sit as generative
counterpoints to the latent JEPA world models above.

| Directory                                   | Paper                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| ------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `paper/navigation-world-models/`            | **NWM ‚Äî Navigation World Models** (Bar et al., CVPR 2025, [2412.03572](https://arxiv.org/abs/2412.03572)): a 1B-param controllable diffusion video world model (linear-in-context CDiT) that plans navigation via MPC+CEM by simulating and scoring imagined trajectories (ATE 1.13 on RECON); the generative counterpoint to latent JEPA world models. _Action-conditioned video / navigation._                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `paper/peva-egocentric-video-prediction/`   | **PEVA ‚Äî Whole-Body Conditioned Egocentric Video Prediction** (Bai et al., 2025, [2506.21552](https://arxiv.org/abs/2506.21552)): autoregressive conditional diffusion transformer predicting egocentric video from whole-body 3D-pose actions (Nymeria), CEM-planned over 16 s rollouts; a pixel-space AC world-model counterpoint to AC-Video-JEPA. _Action-conditioned video world model._                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| `paper/lifting-world-models/`               | **Lifting World Models** (Wang et al., 2026, [2604.26182](https://arxiv.org/abs/2604.26182)): "lifts" a frozen PEVA world model with a small policy mapping low-dim 2D-waypoint high-level actions to joint sequences, so CEM planning runs in an 8-d space (3.8√ó lower joint error vs 48-d joint search). _Egocentric AC world models / hierarchical planning._                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `paper/pointworld/`                         | **PointWorld** (Huang et al., Stanford+NVIDIA, CVPR 2026 Highlight, [2601.03782](https://arxiv.org/abs/2601.03782)): large pre-trained **3D world model** that unifies state and action as **3D point flows** ‚Äî given RGB-D + robot actions (also as point flows over the gripper), a PTv3 backbone + frozen DINOv3 predicts full-scene per-point 3D displacement; the next-frame cloud is decoded by trivial flow addition $x^{(t+1)}=x^{(t)}+d$. ~2M traj / 500 h across Franka + bimanual humanoid; real-time 0.1 s inference; deployed on a real Franka via MPPI-over-SE(3). _Action-conditioned point cloud; the closest analogue to `examples/ac_video_jepa/` in 3D ‚Äî generative, not JEPA, but the state/action recipe is the porting target._                                                                                 |
| `paper/4d-latent-world-model/`              | **4D Latent World Model for Robot Planning** (anon., under review ICLR 2026, [OpenReview](https://openreview.net/pdf?id=d19b200c60835c287e29c6d20ec394f93038c007)): **the closest match to ‚Äúaction-conditioned JEPA + point-cloud decoder‚Äù** ‚Äî encodes multi-view images into a **structured 3D sparse-voxel latent**, predicts future latents conditioned on text/action (Single Dynamics Model for coarse structure + Latent Generator for detail), then **decodes predicted latents into point clouds** (or 3D-Gaussian renders) that feed a goal-conditioned inverse-dynamics head. The decoder is used only to ground latents for the action head, not as the training objective ‚Äî exactly the EB-JEPA philosophy. _Action-conditioned 3D latent + point-cloud decoder; the template for lifting PointWorld into a latent JEPA._ |
| `paper/latent-dyn-pointcloud-registration/` | **Planning with Learned Dynamic Model for Unsupervised Point Cloud Registration** (Yuan et al., IJCAI 2021, [0107](https://www.ijcai.org/proceedings/2021/0107.pdf)): a small, clean, **pre-JEPA (2021) precedent** with every component the hackathon needs ‚Äî encoder-decoder on point clouds (Chamfer reconstruction), a **transformation network** that predicts the latent of the action-transformed cloud $\hat{z}_{X'}=f_\theta^z(z_X, a)$ (the action-conditioned latent predictor), an evaluation network as the reward, and **CEM planning** over actions. Single-object, rigid actions only; adding VICReg/SIGReg would make it a true JEPA. _Action-conditioned point-cloud latent dynamics + CEM; the minimal viable pattern, portable at hackathon scale._                                                               |
| `paper/flowdreamer/`                        | **FlowDreamer** (Guo et al., RA-L 2026, [2505.10075](https://arxiv.org/abs/2505.10075)): two-stage action-conditioned RGB-D world model ‚Äî a U-Net predicts 3D scene flow $f_{t\to t+1}$ from RGB-D + robot action (cross-attention action injection), then a diffusion model renders the future RGB. The **point-cloud baseline variant** uses a FiLM-conditioned MinkowskiNet (sparse 3D conv) as the action-conditioned dynamics predictor. +7% semantic sim, +11% pixel quality, +6% success rate. _Action-conditioned point-cloud dynamics backbone (FiLM-MinkowskiNet) and flow-as-motion-representation, generative not JEPA._                                                                                                                                                                                                  |
| `paper/flow-equivariant-world-models/`      | **Flow Equivariant World Models (FloWM)** (Lillemark, Huang, Zhan, Du, Keller; Harvard/UCSD/CMU, 3 Jan 2026, [2601.01075](https://arxiv.org/abs/2601.01075)): generative world model whose encode‚Üíupdate‚Üíflow‚Üídecode pipeline is **equivariant to Lie-group flows** ‚Äî agent self-motion $T_{a_t}^{-1}$ and external object flow $\psi_1(v)$ compose and act on a velocity-channel latent memory; the decoder renders back to an observation. Long-horizon-stable by a closure property. The closest instantiation of "action = a geometric transform of the latent, decode back to observation," but image/voxel-domain and generative, not JEPA. _Equivariant world-model blueprint for an AC-Point-JEPA: keep the equivariant encode‚Üígroup-action‚Üídecode recurrence, swap the decoder loss for a JEPA L1 + VICReg/SIGReg._          |
| `paper/sen-symmetric-equivariant-wm/`       | **SEN ‚Äî Learning Symmetric Embeddings for Equivariant World Models** (Park et al., KAIST, ICML 2022, [PMLR v162/park22a](https://proceedings.mlr.press/v162/park22a/park22a.pdf)): pre-JEPA equivariant world model that learns a $G$-equivariant transition $T$ in latent space with a **learned** latent group action $\rho_Z(g)$; decoding $\rho_Z(g)\cdot z$ renders the correctly transformed observation (demoed on 3D Teapot SO(3) rotations). The template for "rotate the latent by a group action, decode the rotated observation," but image-domain and uses reconstruction+equivariance loss, not JEPA. _For point clouds with an SE(3)-equivariant encoder, $\rho_Z(g)$ becomes analytic (same $R$ on features) ‚Äî no need to learn it, simplifying the AC-Point-JEPA design._                                            |
